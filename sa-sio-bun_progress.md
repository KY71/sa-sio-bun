# さ小文轉換器 工作進度

## 專案概述
將中文／英文翻譯成台語，再轉換成「さ小文」——一種外觀像日文、唸出來是台語的台灣網路迷因文體。

---

## 已完成

### ✅ 概念確立
- 定義さ小文的本質：**日文皮、國語骨、台語魂**
- 核心反差：視覺像日文，唸出來是台語

### ✅ さ小文四大規則
1. **漢字上限** — 短句最多兩個漢字
2. **漢字選擇** — 中日共用常用字、台語原字、借音有台味的字（如「靠」「叨」）；必須以台羅發音為準，不可直接留原中文字
3. **假名優先** — 盡量平假名，避免片假名
4. **專有名詞保留** — 流行語、新造詞直接保留原字（如「班味」「暈碳」），不計入漢字上限

### ✅ 轉換流程確立
```
中文／英文
    ↓
台語（自然口語）
    ↓
台羅拼音 TL（教育部標準）
    ↓
さ小文
```

## 技術架構（現況）

### 方案：前端 + Vercel 後端代理 + Gemini API
```
使用者輸入
    ↓
HTML／JS 網頁（index.html）
    ↓
Vercel Serverless Function（api/convert.js，後端代理）
  - 帶著伺服器端 Key 去打 Gemini
  - 先查詞彙對照表，動態注入「指定用詞」提示
    ↓
Google Gemini API（gemini-2.5-flash-lite，免費）
    ↓
輸出さ小文（JSON）
```

### 部署流程
1. 程式碼推上 GitHub（repo：KY71/sa-sio-bun）
2. Vercel 連結 GitHub，main 分支一有更新即自動部署
3. 後端機房指定東京（hnd1，見 `vercel.json`）降低延遲

### Gemini API 免費額度
- 每分鐘 15 次請求、每天 1500 次請求
- 改後端代理後，全體使用者共用「同一把 Key」的額度（迷因自用足夠；怕被刷再加流量限制）

### API Key 安全
- ✅ Key 不再寫死於檔案、也不再由使用者輸入
- ✅ 改放 Vercel 環境變數 `GEMINI_API_KEY`，藏在後端，前端原始碼看不到
